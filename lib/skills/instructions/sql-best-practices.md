## Рекомендации по SQL

- **ДГ**: пользователь называет ДГ по имени или номеру → вызови `lookup_dg` → он ищет по Наименованию в справочнике → из результата бери поле Код → в SQL: `dg.[Код] = '<Код>'` (JOIN: `m.[ID_ДГ] = dg.[Код]`).
- При фильтрации по дате предпочитай `CAST(поле AS DATE)` вместо `YEAR()/MONTH()` — это позволяет использовать индексы.
- При JOIN со справочниками всегда используй LEFT JOIN, чтобы не терять записи без связи.
- Если нужно несколько агрегаций по разным разрезам — используй CASE WHEN внутри SUM/COUNT.
- Для процентов используй `CAST(... AS DECIMAL(10,2)) * 100.0 / NULLIF(..., 0)`.
- Если пользователь упоминает город или территорию — вызови `lookup_territory`.
- Если пользователь просит отчёт **по территориям** или «по территориям в Москве/регионе» — строй запрос через `LEFT JOIN [dbo].[Территории] AS ter ON m.[ID_ТерриторияИспользованияТС] = ter.[ID]` и фильтруй/группируй по `ter.[Наименование]`, `ter.[Регион]`, `ter.[ТипТерритории]` или `ter.[ID]`, а не по `m.[Регион]`.
- Если не уверен в точном написании значения (агент, марка, регион) — вызови `list_column_values`.
